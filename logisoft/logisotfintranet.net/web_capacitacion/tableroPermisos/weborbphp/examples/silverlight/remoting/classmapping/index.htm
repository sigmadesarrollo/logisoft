Description: The example demonstrates the following:
  • Automatic type adaptation between the type returned from the server and the type used on the client
  • Server-side class uses Application scope, so only one instance of the class is be created for all connected clients

Key points:
  • The server method returns an instance of Weborb.Examples.PhoneBookRecord. The client application uses ClassMapping.PhoneBookRecord. Since the property names between the two classes match literally, WebORB can provide automatic mapping.
  • Client class ClassMapping.PhoneBookRecord corresponds to the server class Weborb.Examples.PhoneBookRecord. The classes are shown below:
     
    Client C# class Server PHP class 
    namespace ClassMapping
{
    public class PhoneBookRecord
    {
        public string firstName {get;set;}
        public string lastName {get;set;}
        public string phoneNumber {get;set;}
    }
}
         		 
    class PhoneBookRecord
{
    var $firstName;
    var $lastName;
    var $phoneNumber;
}

  • Client and server class properties match literally. Class name/namespace does not need to match the one on the server
